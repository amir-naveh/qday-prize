# Next Step Process

This file defines the iteration loop for advancing the project. Each iteration follows these steps in order:

---

## Iteration Steps

### 1. Review Current State
- Read `current_strategy.md` to understand the current approach and priorities
- Read `log.md` to understand what has been done so far and where we left off

### 2. Draft the Next Step
- Based on the strategy and log, determine the single best next action to advance the project
- The next step must be **granular and concrete** — e.g. create one file, implement one function, run one experiment
- Append the next step to `next_steps.md`, numbered sequentially

### 3. Execute the Next Step
- Carry out the step described in the latest entry of `next_steps.md`
- Do not deviate from the defined step; if it needs to change, update `next_steps.md` first
- Clean up code and any modifications as needed to keep the project clean and well written

### 4. Update the Log
- Append a new dated entry to `log.md` with a high-level description of what was done and any relevant observations

### 5. Update the Strategy
- Review `current_strategy.md` in light of what was just completed
- Update it if the approach, priorities, or understanding has changed
- Leave it unchanged if the strategy still holds

### 6. Review the Process
- Review this file (`next_step_process.md`) and other process files in light of what was just completed
- Make modifications to the process if needed to better achieve the goal

### 7. Commit to GitHub
- Stage all changed and new files
- Commit with a clear, concise message describing what was done in this iteration
- Push to `origin main`

---

## Files

| File | Purpose |
|------|---------|
| `current_strategy.md` | Living strategy document — updated each iteration as needed |
| `next_steps.md` | Numbered list of all planned/executed next steps |
| `log.md` | Chronological log of completed work |
| `next_step_process.md` | This file — the iteration protocol |
