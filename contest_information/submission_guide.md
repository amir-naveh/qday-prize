# Submission Guide

## Deadline
**April 5, 2026** — rolling submissions accepted until closure

## How to Submit
1. Create a **public GitHub repository** containing all required files
2. Email the repository link to: **qdayprize@projecteleven.com**

## Required Files

### 1. Working Code (required)
- All code needed to run your quantum algorithm on real hardware
- Must execute using one of the standard test curves/keys (see `curves_and_keys.md`)
- Must demonstrate successful **private key derivation from the provided public key**

### 2. `brief.pdf` (max 2 pages)
- Explain your ECC key-breaking approach
- Detail the techniques developed or used
- Include diagrams, images, or graphs as needed
- Describe hardware requirements and potential limitations
- Target audience: **quantum algorithm specialists**

### 3. `README.md` (must include)
- Your email address
- Professional background summary
- Key length broken (e.g., "5-bit")
- Quantum computer model used (e.g., "IBM Heron")
- Hardware specifications used
- Access method (e.g., "AWS Braket")
- Detailed step-by-step execution instructions

### 4. Supporting Documentation
- Platform log files from the quantum hardware execution
- Data analysis and charts
- Any other materials relevant to assessment

## Key Rules
- GitHub repository must be **publicly accessible**
- Must demonstrate execution on **actual quantum hardware** (not simulation)
- Submissions will be **made public** — results cannot remain private
- You retain IP ownership, but key findings will be shared for transparency
- Falsified or unverifiable results → **disqualification**

## Verification Process
- An independent panel of **cryptographers and quantum computing experts** reviews submissions
- They attempt independent verification of results
- They verify quantum computer execution and rule compliance

## Teams
- Teams, research groups, and organizations can enter together
- Designate one person as the official point of contact
- No institutional affiliation required
